Bug Fix Leere Katalogseite — _renderCatalog() existierte nicht
Root Cause: _loadTeachers() rief im async Callback _renderCatalog() auf — eine Funktion die nirgends definiert war. Der Fehler wurde vom Browser still verworfen. Die synchrone _render() beim Init zeigte leere Daten (noch nicht geladen) → leere Seite blieb.
Vorher — skiing-catalog.js _loadTeachers()
function _loadTeachers() { AppService.getTeachersWithProfiles(function(err, pairs) { if (err) { ... _renderCatalog(); /* ❌ undefined */ return; } _allTeachers = pairs || []; _applyFilters(); _renderCatalog(); /* ❌ undefined — Browser wirft TypeError, Seite bleibt leer */ }); }
Nachher — korrigiert
function _loadTeachers() { AppService.getTeachersWithProfiles(function(err, pairs) { if (err) { ... _applyFilters(); _buildTabs(); _render(); return; } _allTeachers = pairs || []; _applyFilters(); _buildTabs(); _render(); /* ✅ korrekte Funktion */ _bindCards(); CatalogFilterDrawer.setCount(_filteredTeachers.length); }); }
