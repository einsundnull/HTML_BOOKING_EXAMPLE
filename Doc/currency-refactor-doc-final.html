1 Ziel & Entscheidungen

Ziel: Alle hardcodierten €-Symbole und fmtPrice()-Aufrufe ohne Currency-Code durch korrekte, nutzerabhängige Währungsformatierung ersetzen. Der CurrencyService mit Mockup-Rates (reale Kurse März 2026) war vollständig implementiert — er wurde nur nicht konsequent genutzt.

  • Wallet-Währung: Admin-Seiten immer EUR — alle anderen: displayCurrency des eingeloggten Users

2 Architektur

Statt den CurrencyService-Lookup in jeder Datei zu duplizieren, wurde die gesamte Logik in einem zentralen Helper in ui.js zusammengefasst. Alle anderen Dateien rufen nur noch diesen Helper auf.

/* Vorher — dupliziert in jeder Datei */
var sCur = CurrencyService.getUserCurrency(uid);
if (sCur === 'EUR') return fmtPrice(amount, 'EUR');
var conv = CurrencyService.convertSync(amount, 'EUR', sCur);
if (conv === null) return fmtPrice(amount, 'EUR');
return CurrencyService.format(conv, sCur);

/* Nachher — ein Aufruf überall */
return _fmtForUser(amount, uid);

Die Konversionskette läuft via CurrencyService.warmUp() beim Seitenstart (DOMContentLoaded). Bei der ersten User-Interaktion ist der Rate-Cache garantiert warm.

4 Neue globale Helpers (ui.js)

_fmtForUser(amountEUR, uid)

function _fmtForUser(amountEUR, uid) {
  /* 1. CurrencyService nicht geladen → EUR */
  if (typeof CurrencyService === 'undefined') return fmtPrice(amountEUR, 'EUR');
  /* 2. uid fehlt oder kein Profil → EUR */
  var cur = (uid && typeof ProfileStore !== 'undefined')
    ? CurrencyService.getUserCurrency(uid) : 'EUR';
  /* 3. displayCurrency ist EUR → direkt formatieren */
  if (!cur || cur === 'EUR') return fmtPrice(amountEUR, 'EUR');
  /* 4. Cache kalt → EUR-Fallback */
  var converted = CurrencyService.convertSync(amountEUR, 'EUR', cur);
  if (converted === null) return fmtPrice(amountEUR, 'EUR');
  /* 5. Alles OK → konvertiert formatieren */
  return CurrencyService.format(converted, cur);
}
window._fmtForUser = _fmtForUser;

_getActorUid()

Shared UI-Helpers haben keinen direkten Zugriff auf currentUser. Dieser Helper löst die UID über currentUser oder Auth.current() auf.

function _getActorUid() {
  if (typeof currentUser !== 'undefined' && currentUser) return currentUser.uid;
  if (typeof Auth !== 'undefined' && Auth.current) {
    var u = Auth.current(); return u ? u.uid : null;
  }
  return null;
}
window._getActorUid = _getActorUid;

14 Offene Punkte & Hinweise

1. Redundante lokale Helpers in student.js

_fmtInStudentCur() und _fmtD() sind lokale Inline-Varianten von _fmtForUser() — entstanden vor dem zentralen Helper. Funktional korrekt, aber redundant. Können in einem Cleanup-Schritt durch _fmtForUser(amount, currentUser.uid) ersetzt werden.

2. Wallet-Einzahlung — Eingabewert vs. Anzeigewährung

Das Einzahlungs-Inputfeld zeigt jetzt das Symbol der User-Währung (z.B. ₾). Der eingegebene Betrag wird aber direkt in EUR als Wallet-Guthaben gebucht (AppService.deposit(uid, amount, ...)). Wenn ein User in GEL denkt aber EUR im Wallet hat, ist das semantisch inkonsistent. Für Phase 3 sollte geprüft werden ob AppService.deposit() eine Währungsumrechnung vor der Buchung erhalten soll — oder ob der Input explizit als EUR-Eingabe gekennzeichnet wird.

3. Preis-Filter in skiing-catalog.js

Der Preis-Bereichsfilter (priceMin / priceMax) vergleicht gegen pricePerHalfHour des Teachers, das in Teacher-Währung gespeichert ist. Der Filter-Badge zeigt jetzt die Viewer-Währung — aber der Vergleich selbst ist nicht konvertiert. Das bedeutet: Ein Student mit GEL-Währung sieht den Filter in ₾, aber die Filterlogik vergleicht intern in EUR. Für eine korrekte Filterung müsste pricePerHalfHour des Teachers erst in die Viewer-Währung konvertiert werden.

4. Cache-Warmup Timing

CurrencyService.warmUp() ist async (setTimeout 0). Bei sehr schnellen Interaktionen direkt nach Seitenload kann convertSync() noch null zurückgeben und den EUR-Fallback zeigen. Das ist by design und kein Bug — nach dem ersten Render ist der Cache warm.

Empfehlung für Phase 3
  • Cleanup: _fmtInStudentCur() und _fmtD() in student.js durch _fmtForUser() ersetzen
  • skiing-catalog.js Filterlogik: Teacher-Preise vor Vergleich in Viewer-Währung konvertieren
  • Wallet Einzahlung: Klären ob Eingabe in User-Währung mit Konversion zu EUR gewünscht ist
  • Live-Kurse: currency-provider-mock.js durch currency-provider-live.js ersetzen (Interface identisch)
