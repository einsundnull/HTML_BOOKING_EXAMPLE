student.js — bookSlotWithEscrow Callback (Fix #1 + #2)

✗ Vorher

AppService.bookSlotWithEscrow(sid, stuId, tid, function(e) {
  if (e) Toast.error(e.message || e);
  // kein Re-render, kein Wallet-Refresh
});

✓ Nachher

AppService.bookSlotWithEscrow(sid, stuId, tid, function(e) {
  if (e) { Toast.error(e.message || e); return; }
  renderCalendar();
  renderDaySlots();
  _refreshNavbarWallet(currentUser.uid);
  if (typeof WalletPanel !== 'undefined')
    WalletPanel.refresh(currentUser.uid);
});

SzenarioProblemEmpfohlener Fix
Stornierung direkt aus dem Policy-Dialog (_showCancelPolicyDialog in ui.js) ohne anschließenden Save-Button Der onAction-Callback rendert den Kalender neu, ruft aber nicht WalletPanel.refresh() auf. Refund sichtbar nur im Navbar-Badge, nicht im Panel. In _showCancelPolicyDialog (ui.js ~885) nach dem Navbar-Refresh auch if (typeof WalletPanel !== 'undefined') WalletPanel.refresh(wUid) aufrufen.
requestDepositRefund schlägt fehl wenn depositStatus = 'unpaid' Bei cash_on_site oder nicht bezahltem Deposit: Storno läuft durch, Escrow-Status bleibt unpaid, kein Geld wird bewegt, keine TX geschrieben — kein sichtbarer Fehler. In cancelSlotWithPolicy (app-service.js): wenn Status unpaid, direkt writeCancellationRecord aufrufen statt requestDepositRefund zu versuchen.
