Sync-Vergleich — Welche Views werden nach einem Write aktualisiert?

Das Problem: Änderungen aus dem Day View (neu) werden nicht in Day Panel, All Bookings und My Students reflektiert — und umgekehrt.

Cross-Tab Sync (AppService.onChange)
storage event ✗ ✓ ✓ ✓ ✓ Feuert nur in anderen Tabs — NICHT im selben Tab der schreibt. window.storage event ist browser-intern cross-tab only.
Root Cause
Store.onChange BUG Basiert auf window.addEventListener('storage', ...). Der storage-Event feuert im Browser nur in anderen Tabs, nie im selben Tab. Schreibt man in Tab A, sieht Tab A keinen Event — nur Tab B würde ihn empfangen. Alle Views im selben Tab müssen daher manuell nach jedem Write aktualisiert werden.
Fix — Eine Zeile in jedem _tdv Callback
Day View callbacks Aktuell: _tdvRenderSlots(); renderCalendar();
Soll: _tdvRenderSlots(); renderCalendar(); renderDayPanel(); if (activeTeacherView === 'all-bookings') renderAllBookings(); renderStudentList();

Exakt dasselbe Muster wie AppService.onChange — nur manuell aufgerufen.
