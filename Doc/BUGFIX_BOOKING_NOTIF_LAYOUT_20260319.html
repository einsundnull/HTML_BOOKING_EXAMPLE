chat.js + chat.css

Bug 2 — Details-Button: Stilles Versagen durch innerHTML + getElementById
chat.js — _showBookingNotifOverlay()
Root Cause: Das Overlay-HTML wurde als innerHTML-String mit \"-escaped Attributwerten gebaut (z.B. id=\"cbn-overlay-close-btn\"). Der HTML5-Parser interpretiert \" als wörtliches Anführungszeichen im Attributwert. Dadurch ist die tatsächliche ID im DOM: "cbn-overlay-close-btn" (mit Anführungszeichen), nicht cbn-overlay-close-btn. Das nachfolgende document.getElementById('cbn-overlay-close-btn') gibt null zurück. null.addEventListener() wirft einen TypeError, der vom try/catch stumm geschluckt wird. Die Overlay-Animation startet nie.
Fehlerhafte Ausführungskette (Vorher)
// 1. HTML-String mit \\\" (JS-Escape) wird zu \" im String-Wert overlay.innerHTML = '<button id=\"cbn-overlay-close-btn\">...</button>'; // ^^ ← literale \" im HTML-String // 2. HTML5-Parser sieht: id= dann " → Attributwert beginnt // Wert: "cbn-overlay-close-btn" (mit den Anführungszeichen!) // Tatsächliche DOM-ID: '"cbn-overlay-close-btn"' // 3. getElementById sucht korrekte ID var closeBtn = document.getElementById('cbn-overlay-close-btn'); // → null (ID stimmt nicht überein!) // 4. TypeError: Cannot read properties of null closeBtn.addEventListener('click', closeOverlay); // → TypeError geworfen // 5. try/catch fängt ab, _debugLog schreibt in console.error // 6. requestAnimationFrame(overlay.classList.add) wird NIE erreicht // 7. Overlay erscheint nicht — kein sichtbarer Fehler für den User
Fix: Vollständiger Umbau von _showBookingNotifOverlay() auf createElement / appendChild. Alle DOM-Referenzen (closeBtn, walletBtn) werden als JavaScript-Variablen gehalten. Keine IDs für interne Buttons. Event-Listener direkt auf die Variablen gesetzt — keine getElementById-Suche nach innerHTML-Injection.
❌ Vorher — innerHTML + getElementById
overlay.innerHTML = '<button id=\"cbn-overlay-close-btn\">…</button>'; // ↑ id enthält Anführungszeichen im DOM! document.getElementById('cbn-overlay-close-btn') .addEventListener('click', closeOverlay); // ↑ null → TypeError → alles bricht ab
✅ Nachher — createElement + direkte Referenz
var closeBtn = document.createElement('button'); closeBtn.className = 'cbn-overlay-close'; closeBtn.setAttribute('aria-label', ...); closeBtn.innerHTML = '<svg>...</svg>'; // ↑ nur SVG-Inhalt per innerHTML, keine IDs closeBtn.addEventListener('click', closeOverlay); // ↑ direkte Variable → kein getElementById nötig header.appendChild(closeBtn); panel.appendChild(header); overlay.appendChild(panel);
Architektur-Regel (abgeleitet aus diesem Bug)
Regel: HTML-Strings mit \"-escaped Attributwerten dürfen nicht für dynamisch erstellte DOM-Elemente verwendet werden, wenn anschließend per getElementById oder querySelector('[id=...]') auf diese Elemente zugegriffen werden soll.
PatternSicher?Warum
el.innerHTML = '...id=\"foo\"...';
document.getElementById('foo')		 Unsicher HTML5-Parser kann \" als wörtliches Zeichen im Attributwert interpretieren → ID im DOM enthält Anführungszeichen → getElementById findet null
var btn = document.createElement('button');
btn.id = 'foo';
btn.addEventListener(...);		 Sicher ID wird als JS-Property gesetzt — kein HTML-Parser involviert. Event-Listener direkt auf Variable — kein getElementById nötig.
el.innerHTML = '...';
el.querySelector('.class-name')		 Sicher Klassen-Selektoren mit \"-Escaping funktionieren zuverlässig. Wie bei anderen Service-Messages im Projekt (student_request, booking_request).
Inline-SVG per innerHTML Sicher SVG-Inhalt enthält keine IDs die nachher per JS gesucht werden. Keine getElementById-Abhängigkeit.

Korrekter Ausführungsfluss nach dem Fix
1
User klickt «Details ›» in der Chat-Card
detailsBtn.addEventListener('click', ...) — Button-Referenz ist JS-Variable, kein DOM-Lookup nötig. e.stopPropagation() verhindert Bubbling.
2
_showBookingNotifOverlay(snap, actorUid) aufgerufen
Existierendes Overlay wird entfernt (Duplicate-Guard). Alle DOM-Elemente via createElement gebaut — kein HTML-Parser involviert.
3
overlay.appendChild(panel) → document.body.appendChild(overlay)
Overlay ist im DOM. closeBtn und walletBtn sind bereits bekannte Variablen — Event-Listener werden direkt gebunden.
4
requestAnimationFrame → overlay.classList.add('is-open')
CSS-Transition (opacity + translateY) animiert das Bottom-Sheet herein. User sieht Overlay mit Slot-Details und «Im Wallet anzeigen»-Button.
5
User klickt «Im Wallet anzeigen» oder ✕ oder Backdrop
closeOverlay() entfernt is-open, nach 300ms wird das Element aus dem DOM entfernt. ESC-Listener wird ebenfalls sauber entfernt.
