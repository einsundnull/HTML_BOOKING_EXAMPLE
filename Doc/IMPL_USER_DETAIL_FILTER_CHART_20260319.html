4
Call-Pipeline — Jeder Filter-Auslöser
DOM Event
input / change / click
User ändert einen Filter-Wert (Text, Datum, Betrag, Dropdown-Auswahl) oder klickt „Anwenden" im Personen-Drawer.
↓
user-detail.js
Event-Handler in _bindFilterBar()
Schreibt den neuen Wert in _fs (Filter-State-Objekt) bzw. _selParties (Personen-Map). Ruft danach _refilter() auf.
↓
user-detail.js
_refilter()
Ruft _applyFilters(_allTxs) → gibt gefilterte TX-Liste zurück. Gibt diese an _renderTransactions(filtered), _renderChart(filtered) und _updateFilterUI(count).
↓
user-detail.js
_applyFilters(txs)
Wendet alle 8 Filter sequenziell an. Gibt Array zurück. Liest aus _fs + _selParties.
↓
user-detail.js
_renderTransactions(txs)
Baut TX-Liste neu auf. Jedes <li> via _buildTxItem(tx) mit Click → WalletCore.showTxDetail(tx, _uid).
user-detail.js
_renderChart(txs)
Zeichnet SVG-Chart neu. Braucht mind. 2 TXs mit balance-Feld. Zeigt nur gefilterte Datenpunkte.
user-detail.js
_updateFilterUI(count)
Setzt Zähler-Text (#ud-filter-count), zeigt/versteckt Reset-Button, aktualisiert Personen-Button-Label mit aktiver Anzahl.
5
Alle neuen Funktionen
FunktionSignaturBeschreibung
_applyFilters(txs[]) → tx[]Alle 8 Filter sequenziell. Liest _fs + _selParties.
_isFiltered() → booleanTrue wenn mindestens ein Filter aktiv ist. Steuert Reset-Button-Sichtbarkeit.
_refilter() → voidZentrale Orchestrierung: applyFilters → renderTransactions + renderChart + updateUI.
_updateFilterUI(count) → voidZähler, Reset-Button, Party-Button-Label.
_bindFilterBar() → voidRegistriert alle Event-Listener. Aufgerufen einmalig in _load().
_bindUdDropdown(ddId, onChange) → voidBindet Custom Dropdown (Typ + Status). Gleiche Logik wie admin.js _bindAdminDropdowns.
_resetUdDropdown(ddId, placeholder) → voidSetzt Dropdown auf ersten Eintrag zurück.
_closeAllUdDropdowns() → voidSchließt alle offenen Filter-Dropdowns.
_openPartyDrawer() → voidErstellt Bottom-Sheet mit Checkboxen der verbundenen Personen. Schreibt Ergebnis in _selParties.
_loadParties() → voidFüllt _parties[] via getSelectionsByTeacherSync oder getSelectionsByStudentSync.
_renderChart(txs[]) → voidSVG-Liniendiagramm: X = Datum, Y = Balance. Grid-Linien, Area-Fill, Dots mit Tooltip.
6
State-Variablen (Modul-global)
VariableTypInhalt
_allTxsTX[]Alle TXs des Users, geladen einmalig, nach Datum absteigend sortiert. Wird nie mutiert.
_fsObjectFilter-State: { txid, type, status, from, to, amtMin, amtMax }. Wird bei Reset komplett neu gesetzt.
_parties[{uid, name}]Verbundene Personen. Einmalig in _loadParties() befüllt.
_selParties{ uid: boolean }Aktive Personen-Auswahl aus dem Drawer. Key = uid, Value = true wenn ausgewählt.
7
Chart — Technische Details
AspektImplementierung
TypLiniendiagramm mit Area-Fill — reines SVG, kein Framework
X-Achsetx.createdAt — gleichmäßig verteilt (linear, nicht zeitproportional)
Y-Achsetx.balance — Kontostand nach der TX. Min/Max dynamisch.
Grid4 horizontale Linien mit Y-Labels in Euro
DotsJeder Datenpunkt als Kreis mit SVG <title>-Tooltip (Typ + Datum + Saldo)
ReaktivitätBei jedem _refilter()-Aufruf wird SVG komplett neu gezeichnet
Min. DatenpunkteMind. 2 TXs mit balance-Feld. Sonst: .ud-chart-wrap.is-hidden
Breitewrap.offsetWidth — responsive, passt sich dem Container an
