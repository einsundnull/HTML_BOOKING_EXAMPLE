2 Datenmodell — Selection Schema erweitert store.js · app_selections
Warum auf der Selection? Die Beziehung „Lehrer ↔ Schüler hat einen eigenen Preis" gehört auf die Selection — nicht auf das Profil des Lehrers und nicht auf den Slot. Das Schema bleibt flach und ist direkt Firestore-kompatibel.
store.js — app_selections Schema (vorher → nachher)
/* Vorher */ { studentId: 's115', teacherId: 't1' } /* Nachher — priceOverride optional */ { studentId: 's115', teacherId: 't1', priceOverride: 45 } /* priceOverride: null → kein Override, Standardpreis gilt */ /* priceOverride: 0 → gültig (Nullpreis ist möglich) */ /* priceOverride: fehlt → kein Override (Legacy-Kompatibilität) */
store.js : Zeile 460 — Selections.update()
/* Merge patch into an existing selection */ update: function(sid, tid, patch) { var list = this.all().map(function(s) { if (s.studentId !== sid || s.teacherId !== tid) return s; var u = {}; for (var k in s) u[k] = s[k]; for (var p in patch) u[p] = patch[p]; return u; }); _save(KEYS.SELECTIONS, list); }
3 Preis-Prioritätskette bei Buchung adapter-localstorage.js : bookSlot() · Zeile 412
Slot-Preis wird zum Buchungszeitpunkt eingefroren (slot.price). Spätere Änderungen am Profil oder am priceOverride wirken sich nur auf neue Buchungen aus — bestehende Buchungen bleiben unverändert.
1. Selection.priceOverride — falls für diesen Schüler gesetzt (nicht null, nicht leer)
2. Teacher-Profil pricePerHalfHour — Standardpreis, falls kein Override

4 AppService — neue öffentliche Methoden app-service.js
MethodeSignaturRückgabeZweck
updateSelection (studentId, teacherId, patch, callback) callback(err, true) Schreibt priceOverride (oder andere Felder) auf eine Selection. patch = { priceOverride: null } setzt zurück auf Standardpreis.
getStudentPriceForTeacherSync (studentId, teacherId) number Gibt den effektiven Preis zurück: priceOverride wenn gesetzt, sonst Profil-Standardpreis. Wird in der UI für die Anzeige verwendet.
Beide Methoden sind in den Exports registriert: updateSelection im Async-Block, getStudentPriceForTeacherSync im Sync-Block von AppService. Der Adapter-Firestore muss beide Methoden ebenfalls implementieren, bevor auf Produktion gewechselt wird.
