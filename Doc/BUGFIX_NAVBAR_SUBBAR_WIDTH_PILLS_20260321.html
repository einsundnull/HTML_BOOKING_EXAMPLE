Sub-Bar kollabiert auf Mobile & Desktop-Pills stacken
Zwei unabhängige CSS-Fehler im gleichen Render-Pfad. (1) .navbar-subbar fehlte width: 100% — als Kind von <body> (kein Flex-/Grid-Container) kollabierte der Sub-Bar auf 0px Breite: weißer Streifen, keine Buttons sichtbar. (2) .navbar-selects-desktop hatte kein display: flex — die beiden Pill-Buttons (Währung + Sprache) stackten vertikal statt nebeneinander.

Implementierter Fix

1 Datei geändert · navbar.css
Datei Selector Hinzugefügte Eigenschaft Begründung
css/navbar.css .navbar-subbar width: 100% Ohne explizite Breite kollabiert ein display: flex Element als Kind von <body> auf seine Inhaltsbreite (0px bei leerem/kollabiertem Inhalt). width: 100% streckt es auf volle Viewport-Breite.
css/navbar.css .navbar-subbar-inner width: 100% margin: 0 auto benötigt eine definierte Breite auf dem Element selbst, damit der Auto-Margin zentrieren kann. Ohne width: 100% kollabiert auch der innere Container.
css/navbar.css .navbar-selects-desktop display: flex; align-items: center; gap: var(--sp-2) Der <div>-Wrapper hatte keine explizite display-Regel. Browser-Default ist block → Kind-Elemente (.navbar-select-wrap) stackten vertikal. display: flex legt sie nebeneinander. Die bestehende Media-Query @media (max-width: 768px) { display: none } überschreibt dies korrekt auf Mobile.
js/app-config.js APP_VERSION '2026-03-21-30' → '2026-03-21-31' Cache-Buster für navbar.css.
css/navbar.css — Bug 1: Sub-Bar width fix
