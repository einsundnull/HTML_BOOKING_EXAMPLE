  getEscrowBySlot(slotId, cb)         → Escrow | null
  getEscrowsByStudent(uid, cb)        → Escrow[]
  getEscrowsByTeacher(uid, cb)        → Escrow[] (mit Schülernamen)
  payDeposit(escrowId, cb)            → { escrow, transaction } — unpaid→held
  requestDepositRefund(escrowId, cb)  → void — held→refund_requested
  releaseDeposit(escrowId, cb)        → { wallet, transaction } — →refunded (Schüler)
  forfeitDeposit(escrowId, cb)        → { wallet, transaction } — →forfeited (Lehrer)
  confirmLesson(escrowId, cb)         → { escrow, transaction } — →released
  adminReleaseEscrow(escrowId, adminUid, cb) → void — Admin-Override, 2 TX

ESCROW KOMBINIERT (UI sollte diese nutzen, nicht die Primitiven)
  bookSlotWithEscrow(slotId, studentId, teacherId, cb)
    Atomare Buchung: bookSlot + createEscrow + payDeposit.
    Rollback bei Fehler (cancelSlotBooking wenn Escrow/Deposit fehlschlägt).
    cb: fn(err, { escrow, transaction|null })

  cancelSlotWithPolicy(slotId, actorRole, cb)
    Atomare Stornierung: cancelSlotBooking + Escrow-Aktion je Tier.
    actorRole: 'student'|'teacher'|'admin'
    cb: fn(err, { policy, transaction|null })
    Tier-Logik:
      teacher_cancel / full_refund → requestDepositRefund + releaseDeposit
      forfeit / partial (MVP)      → requestDepositRefund + forfeitDeposit
      no_escrow                    → nur writeCancellationRecord (kein Geld)

  calcCancellationPolicy(slotId, actorRole, cb)
    Berechnet was bei Stornierung passiert (OHNE zu stornieren).
    cb: fn(err, { tier, noticeHours, depositStatus, depositAmount,
                  refundAmount, forfeitAmount, escrowId, paymentMode })
    Tiers: 'full_refund'|'partial'|'forfeit'|'teacher_cancel'|'no_escrow'

SONSTIGES
  onChange(fn)                        → Cross-Tab-Sync Listener

3F. Escrow-Datenschema (localStorage key: app_escrow)
────────────────────────────────────────────────────────────────────────────────

Felder pro Escrow-Objekt:
  escrowId          'esc_<timestamp>'
  slotId            Referenz auf app_slots
  studentId
  teacherId
  depositAmount     EUR
  depositType       'fixed' | 'percent'
  depositPercent    z.B. 20 (null wenn fixed)
  fullAmount        Gesamtpreis der Stunde
  negotiatedAmount  null oder abweichender Preis
  paymentMode       'instant' | 'cash_on_site'
  requiresDeposit   Boolean
  depositStatus     'unpaid'|'held'|'refund_requested'|'refunded'|'forfeited'|'released'
  fullPaymentStatus 'unpaid'|'paid'|'verified'
  slotDate          'YYYY-MM-DD' (direkt gespeichert, kein Slot-Lookup nötig)
  slotTime          'HH:MM'
  depositPaidAt
  releasedAt
  createdAt

Deposit-Status-Flow:
  unpaid
    → held              (payDeposit)
      → refund_requested (requestDepositRefund)
        → refunded       (releaseDeposit)   — Deposit → Schüler
        → forfeited      (forfeitDeposit)   — Deposit → Lehrer
      → released         (confirmLesson)    — Stunde bestätigt

3G. Transaktions-Schema (localStorage key: app_transactions)
────────────────────────────────────────────────────────────────────────────────

Felder:
  txId            'tx_<timestamp>_<random>'
  uid             Wallet-Inhaber
  type            'deposit'|'withdrawal'|'escrow_hold'|'escrow_release'
                  |'escrow_refund'|'cancellation'
  amount          EUR (negativ bei Abzug)
  balance         Saldo nach TX
  description     Lesbarer Text: 'Stornierung: Name, Datum, Tier'
  status          'completed'|'pending'|'failed'
