A — buildAvatarHTML() · globale Hilfsfunktion

Einmalig in ui.js definiert und als window.buildAvatarHTML exportiert. Wird von teacher.js, skiing-catalog.js und zukünftig chat.js wiederverwendet — kein Codeduplikat.

/**
 * buildAvatarHTML(uid, opts)
 * opts: { size: 'sm'|'md'|'lg', role: 'teacher'|'student'|'admin' }
 * Gibt HTML-String zurück: <img> wenn Foto, sonst <div> mit Initialen.
 */
function buildAvatarHTML(uid, opts) {
  var photo    = ProfileStore.getPhoto(uid);
  var name     = ProfileStore.getDisplayName(uid);
  var initials = _computeInitials(name);
  if (photo) {
    return '<img class="avatar-img avatar-' + size + '" src="...';
  }
  return '<div class="avatar-initials avatar-' + size + ... initials + '</div>';
}

B — showProfileSheet(uid) · globales Bottom-Sheet

Öffnet ein animiertes Bottom-Sheet mit den Profildaten eines beliebigen Nutzers (Schüler oder Lehrer). Verwendet ProfileStore.getOrDefault(uid) — kein AppService-Aufruf nötig. Renutzt pv-* CSS-Klassen aus profile.css (jetzt auf teacher/student geladen).

D — Lehrer-Profil-Link (Student → Katalog)

In _buildSummaryHTML() in skiing-catalog.js wird ein neuer <a class="tc-profile-btn"> eingefügt — zwischen Chat-Button und Expand-Button. Navigiert direkt zu profile-view.html?uid={uid}.

Button-Reihenfolge in der Karte (rechts)

1. tc-chat-btn — Chat öffnen
2. tc-profile-btn NEU — profile-view.html öffnen
3. tc-expand-btn — Karte aufklappen
Architektur-Hinweis: tc-profile-btn ist ein <a>-Element (nicht button), da es zu einer neuen Seite navigiert — semantisch korrekt, kein JavaScript nötig, kein Event-Listener.

Architektur-Entscheidungen

EntscheidungBegründung
buildAvatarHTML in ui.js, nicht in profile.js oder teacher.jsui.js ist auf allen Seiten geladen. Vermeidet Duplikat in teacher.js und skiing-catalog.js
showProfileSheet in ui.js statt Modal.show()Modal.show() verwendet ein festes Layout. Bottom-Sheet passt besser auf Mobile. Wiederverwendet wtx-detail-* Klassen (bereits auf teacher/student geladen)
Kein neues HTML-File für Schüler-ProfilProfileStore.getOrDefault() liefert alle Daten synchron. Kein Netzwerkaufruf nötig
