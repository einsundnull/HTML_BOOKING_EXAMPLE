2 materialiseWeek — Implementierung adapter-localstorage.js

Kernfunktion: Recurring Rules für eine Woche in konkrete Slots materialisieren. Konvertiert rule.time (Lokalzeit) → UTC pro Datum. Schützt Slots mit Konfiguration vor dem Purge.

2.1 Purge-Schutz

Nur rule-gedeckte Plain-Slots werden ersetzt. Folgende Slots niemals purgen:

  • Gebuchte Slots (studentId, confirmedAt, bookedAt)
  • Timeout / Disabled — explizite Lehrer-Aktion
  • Slots mit visibility, visibilityList, excludeNewStudents, groupMax, autoPromotedStudents
  • Manual-Slots ohne passenden Rule-Key (date|utcTime nicht in recurringKeys)
/* Purge-Bedingung — gibt true zurück = Slot bleibt erhalten */
var purged = allSlots.filter(function(s) {
  if (s.teacherId !== teacherId) return true;
  if (!weekDateStrs[s.date]) return true;
  if (_pendingIds.indexOf(s.slotId) !== -1) return true;
  if (s.studentId || s.confirmedAt || s.bookedAt) return true;
  if (s.status === 'timeout' || s.baseStatus === 'timeout') return true;
  if (s.status === 'disabled' || s.baseStatus === 'disabled') return true;
  /* Slots mit Visibility/Group-Konfiguration nie purgen */
  if (s.visibility && s.visibility !== 'public') return true;
  if (s.visibilityList && s.visibilityList.length) return true;
  if (s.excludeNewStudents) return true;
  if (s.autoPromotedStudents && s.autoPromotedStudents.length) return true;
  if (s.groupMax && s.groupMax > 1) return true;
  /* Manual-Slot ohne passende Rule → behalten */
  var slotKey = s.date + '|' + s.time;
  if (!recurringKeys[slotKey]) return true;
  return false; /* rule-backed plain slot → purgen und neu erstellen */
});

2.2 UTC-Konvertierung pro Datum (DST-korrekt)

var teacherTZ = TimezoneService.getUserTimezone(teacherId);

for (var i = 0; i < rules.length; i++) {
  var rule = rules[i];
  var dateStr = weekDates[rule.dayOfWeek].toISOString().slice(0, 10);

  /* Lokalzeit → UTC für dieses spezifische Datum (DST-korrekt) */
  var conv = TimezoneService.localToUtc(rule.time, dateStr, teacherTZ);
  var utcTime = conv.utcTime;
  var utcDate = dateStr;

  /* Mitternachts-Überschreitung — Slot gehört zum Nachbar-UTC-Datum */
  if (conv.dateOffset !== 0) {
    var adj = new Date(dateStr + 'T12:00:00');
    adj.setDate(adj.getDate() + conv.dateOffset);
    utcDate = adj.toISOString().slice(0, 10);
  }

  var key = teacherId + '|' + utcDate + '|' + utcTime;
  if (!existSet[key]) {
    /* Visibility-Config vom Original-Slot übernehmen (falls vorhanden) */
    var orig = originalByKey[utcDate + '|' + utcTime];
    var newSlot = { date: utcDate, time: utcTime, _utc: true, ... };
    if (orig) {
      if (orig.visibility) newSlot.visibility = orig.visibility;
      if (orig.visibilityList) newSlot.visibilityList = orig.visibilityList;
      if (orig.groupMax) newSlot.groupMax = orig.groupMax;
      // ... weitere Felder
    }
    purged.push(newSlot);
  }
}
3 flushGridPending — Regel-Speicherung teacher.js

Wenn der Lehrer Slots im Grid speichert, wird für jeden Slot eine Recurring Rule erstellt. Die Rule erhält Lokalzeit — nicht UTC.

var time      = parts[1];   /* UTC — für Slot-Operationen */
var localTime = time;       /* Fallback */

/* UTC → Lokal für Rule-Speicherung */
if (typeof TimezoneService !== 'undefined') {
  var tz  = TimezoneService.getUserTimezone(currentUser.uid);
  var loc = TimezoneService.utcToLocal(time, date, tz);
  if (loc && loc.localTime) localTime = loc.localTime;
}

/* Rule: Lokalzeit. Slot: UTC mit _utc:true */
AppService.createRecurring(currentUser.uid, dow, localTime, cb);
AppService.createSlot({ time: time, _utc: true, ... }, cb);

Beim Löschen einer Rule (disabled-Aktion) ebenfalls Lokalzeit übergeben:

AppService.deleteRecurringByDayTime(uid, dow, localTime, cb);
4 renderGridTable — Recurring-Check teacher.js

Der Recurring-Check beim Rendern des Grids vergleicht gegen localLabel (die angezeigte Lokalzeit der Zeile), nicht gegen UTC. Konsistent mit dem Rule-Speicherformat.

/* localLabel = "HH:MM" lokal (z.B. "12:30" für UTC+4 Lehrer) */
var slot      = AppService.slotExistsSync(uid, cellDateStr, cellUtcTime);
var recurring = AppService.recurringExistsSync(uid, dayOfWeek, localLabel);

Beim Timeout-Toggle (onGridCellClick):

var localT = cell.dataset.localtime || time;
var rec2   = AppService.recurringExistsSync(uid, dayOfWeek, localT);
5 One-Time Migration teacher.js · app-config.js

Konvertiert bestehende naive Slot-Zeiten (Lokalzeit) → UTC. Läuft einmalig nach erstem Deploy.

Vorher — kaputt

IIFE in app-config.js zur Parse-Zeit ausgeführt. AppService ist undefined → Migration läuft nie.

/* FALSCH — AppService undefined */
(function() {
  if (typeof AppService !== 'undefined') {
    AppService.migrateTimesToUtc(...);
  }
})(); // ← läuft SOFORT, defer noch nicht geladen
Nachher — korrekt

Migration im window 'load' Handler in teacher.js — nach allen defer-Scripts.

/* KORREKT — teacher.js load handler */
window.addEventListener('load', function() {
  /* AppService garantiert bereit */
  AppService.migrateTimesToUtc(function(err) {
    if (err) console.warn(err);
  });
  _initApp();
});

Migration-Logik (migrateTimesToUtc in adapter-localstorage.js)

  • Prüft Flag app_tz_migrated_v1 in localStorage — läuft nur einmal
  • Überspringt Slots mit slot._utc = true (bereits migriert / vom App erstellt)
  • Konvertiert naive Zeiten via localToUtc(slot.time, slot.date, teacherTZ)
  • Passt slot.date bei Mitternachts-Überschreitung an
  • Setzt slot._utc = true nach Konvertierung
_utc-Flag Semantik: slot._utc = true bedeutet „slot.time ist UTC, keine Migration nötig". Wird gesetzt von: createSlot (immer), materialiseWeek (neue Slots), Migration (nach Konvertierung). Bleibt persistent im localStorage.
6 DST-Verhalten

Jede Zelle im Grid ruft localToUtc(localLabel, cellDateStr, tz) mit dem eigenen Datum auf. Dadurch bekommt jede Spalte den richtigen UTC-Offset für ihr Datum — auch in DST-Übergangwochen.

DatumTimezoneOffsetLokalzeit "12:30"UTC-Slot
Samstag, 28. März 2026Europe/BerlinUTC+1 (Winter)12:30 Lokal11:30 UTC
Sonntag, 29. März 2026Europe/BerlinUTC+2 (Sommer, ab 2:00)12:30 Lokal10:30 UTC
Montag, 30. März 2026Europe/BerlinUTC+2 (Sommer)12:30 Lokal10:30 UTC
Ergebnis: Recurring Rule time = "12:30" (Lokal) erzeugt in der DST-Übergangswoche Slots bei 11:30 UTC (Sa) und 10:30 UTC (So+). Der Lehrer sieht immer 12:30 Uhr seiner Lokalzeit — egal welche Woche.
