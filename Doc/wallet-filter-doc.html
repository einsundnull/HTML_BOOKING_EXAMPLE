Wallet Filter

Vollständige Dokumentation · BookingSystem · 18.03.2026

1 — Überblick

Die Wallet-Transaktionsliste unterstützt jetzt umfassende Filteroptionen. Alle Filter arbeiten clientseitig auf dem einmal geladenen Transaktions-Array — kein zusätzlicher API-Aufruf bei Filteränderungen.

Architektur-Prinzip: AppService.getTransactions(uid, cb) wird einmalig aufgerufen und speichert alle TXs in _allTxs. Jede Filteränderung ruft refilterAndRender() auf, das _applyWalletFilters(_allTxs) aufruft und das Ergebnis an renderHistory() übergibt.

FilterUI-ElementTX-FeldLogik
TX-Typ Toggle-Chips (Einfachauswahl) tx.type 'cancellation' matcht auch teacher_cancel
Datum Von Date-Input tx.createdAt.slice(0,10) >= dateFrom (ISO-String Vergleich)
Datum Bis Date-Input tx.createdAt.slice(0,10) <= dateTo
Betrag Min Number-Input Math.abs(tx.amount) >= amtMin — abs() damit negative Beträge auch matchen
Betrag Max Number-Input Math.abs(tx.amount) <= amtMax
Gegenseite Native Select tx.relatedUid, tx.meta.teacherId, tx.meta.studentId Lehrer sieht Schüler-Dropdown, Schüler sieht Lehrer-Dropdown. Nur sichtbar wenn >1 Gegenseite vorhanden
Sortierung Sort-Button (↑/↓) tx.createdAt Toggle zwischen neueste-zuerst (default) und älteste-zuerst
Anzahl-Anzeige Text-Badge — "3 von 47 Transaktionen" — nur wenn Filter aktiv
Reset Button (nur wenn Filter aktiv) — Alle Filter auf Default, UI zurücksetzen

5 — Filter-Pipeline

1
AppService.getTransactions(uid, cb) → speichert in _allTxs (master copy, nie mutiert)
↓
2
refilterAndRender() → ruft _applyWalletFilters(_allTxs)
↓
3
_applyWalletFilters(txs): type → dateFrom → dateTo → amtMin → amtMax → party → sort
↓
4
renderHistory(filtered) → DOM neu aufgebaut
↓
5
_updateFilterCount(filtered, total) → Anzahl-Badge
_updateResetBtn() → Reset-Button ein/aus

var _filters = {
  type:     'all',      // TX-Typ oder 'all'
  dateFrom: '',         // ISO date string 'YYYY-MM-DD' oder ''
  dateTo:   '',         // ISO date string 'YYYY-MM-DD' oder ''
  amtMin:   '',         // string (parseFloat bei Verwendung)
  amtMax:   '',         // string (parseFloat bei Verwendung)
  party:    'all',      // UID oder 'all'
  sortAsc:  false        // false = neueste zuerst (default)
};
