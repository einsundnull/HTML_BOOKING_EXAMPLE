It renders #day-save-summary — a position:fixed element — regardless of which view is active.
So when the old day-slots accordion is open AND the student stages a booking in it,
the summary panel appears floating over the old accordion, not the day view.

Fix — merge into one dismiss function
Fix 1 Replace both discardDayChanges() and _sdvDiscardPending() with a single

shows Toast, closes day view if needed.
Fix 2 #day-dismiss-btn always calls discardAllPending() — no if (_sdvOpen) branching needed.
