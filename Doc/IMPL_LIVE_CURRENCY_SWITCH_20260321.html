3 Währungs-Prioritätskette — getUserCurrency(uid) currency-service.js · SSOT
Eine Funktion für alle Stellen: CurrencyService.getUserCurrency(uid) ist der einzige erlaubte Einstiegspunkt um die aktuelle Anzeigewährung zu ermitteln. Sie deckt alle drei Fälle ab: eingeloggt, Gast, erster Aufruf.

4 Persistenz-Architektur — GuestSettings store.js → adapter → app-service
Warum GuestSettings? Alle anderen localStorage-Collections (Slots, Users, etc.) sind ausschließlich via AppService-Layer zugänglich. GuestSettings folgt demselben Muster — die Währungswahl eines Gastes ist eine eigene Collection, kein Anhang an eine andere Entität.
Schema
Key: app_guest_settings
Wert: { currency: 'USD' }

Flaches Objekt. Kein Array. Firestore-kompatibel als einzelnes Document in einer settings-Collection.
Schicht-Stack
Store.GuestSettings.setCurrency(code)
↑ LocalStorageAdapter.setGuestCurrency(code, cb)
↑ AppService.setGuestCurrency(code, cb)
↑ CurrencyService.setGuestCurrency(code, uid)
↑ Navbar._bindCurrencyDropdown()
Sync-Lesepfad (dokumentierte Ausnahme): getUserCurrency(uid) liest Store.GuestSettings direkt (synchron), weil alle Render-Funktionen (_buildCard, etc.) synchron arbeiten. Dies ist die einzige Stelle im Consumer-nahen Code, die den Store direkt liest. Beim Wechsel auf Firestore wird dieser Sync-Read durch einen warm gecachten Wert ersetzt.

6 Preis-Konvertierungslogik — zwei Betragstypen currency-service.js · SSOT
BetragstypGespeichert inKonvertierungsfunktionEinsatz
Wallet / Transaktionen / Deposits Immer EUR (Systemwährung) _fmtForUser(amountEUR, uid) wallet-panel.js, wallet-core.js, Buchungsdialoge in ui.js
Lehrerpreis (pricePerHalfHour) Beliebige Währung (priceCurrency) convertSync(price, priceCurrency, viewerCurrency) skiing-catalog.js, profile.js, ui.js (Profile-Sheet), student.js
Nicht redundant: Die beiden Konvertierungspfade sind bewusst getrennt. _fmtForUser() erwartet EUR als Input und konvertiert in die Viewer-Währung. convertSync() konvertiert von einer beliebigen Quellwährung. Ein Lehrer kann seinen Preis z.B. in CHF eingeben — _fmtForUser() würde hier falsch rechnen.
