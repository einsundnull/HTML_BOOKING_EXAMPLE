Warum _uid und nicht tx.uid?
Beide Werte sind in diesem Kontext gleich: _uid ist der User auf der Admin-Detailseite, und alle geladenen TXs gehören diesem User (tx.uid === _uid).

Dennoch wird _uid verwendet, weil WalletCore._showTxDetail() uid für drei Zwecke nutzt: (1) Währungsformatierung via _fmtForUser(amount, uid), (2) Related-TX Lookup via AppService.getTransactions(uid, cb), (3) Bestimmung des Kontakt-Empfängers im TX-Inquiry-Dialog. Alle drei Fälle erwarten den Wallet-Inhaber — das ist _uid.
