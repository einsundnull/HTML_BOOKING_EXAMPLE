FunktionTX-TypBeschreibungStatus

writeBookingRecord(slotId, stuId, tId, escrow, cb) booking Schreibt je einen TX beim Schüler UND beim Lehrer für jede Buchung — auch bei cash_on_site ohne Deposit. Enthält alle Buchungsdetails im meta-Objekt: fullAmount, depositAmount, depositType, depositPercent, paymentMode, requiresDeposit, bookedAt, escrowId. ✓ Done

confirmLesson() — erweitert lesson_confirmed Schreibt zusätzlich zum Lehrer-TX (escrow_release) jetzt auch einen lesson_confirmed-TX beim Schüler. Vorher hatte der Schüler keine sichtbare Buchungsbestätigung in seiner Historie. ✓ Done
getAllTransactions(opts, cb) — Admin-Funktion: gibt ALLE TXs zurück, filterbar nach uid, types[], dateFrom, dateTo, slotDateFrom, slotDateTo, teacherId, studentId, counterpartId. Basis für den Admin-Audit-Screen. ✓ Done
getBookingHistory(uid, opts, cb) — User-Funktion: gibt alle buchungsrelevanten TXs für einen User zurück (booking, move, cancellation, escrow_hold, refund, escrow_release, teacher_cancel, lesson_confirmed). Filterbar nach dateFrom/dateTo/counterpartId. ✓ Done
