2 — Escrow-Zustandsmaschine
DEPOSIT-STATUS-FLOW:
  unpaid ──payDeposit──► held ──requestDepositRefund──► refund_requested ──releaseDeposit──► refunded (Schüler)
                                                                         └──forfeitDeposit──► forfeited (Lehrer)
  held ──confirmLesson──► released (Lehrer)

STORNO-TIER:
  >48h Vorankündigung → full_refund → Schüler bekommt alles zurück
  24–48h Vorankündigung → partial → 50% Deposit an Lehrer (MVP: alles an Lehrer)
  <24h Vorankündigung → forfeit → volles Deposit an Lehrer
  Lehrer storniert → teacher_cancel → immer volle Rückerstattung Schüler
  kein Escrow → no_escrow → seit Fix D: writeCancellationRecord (kein Geld, aber Protokoll)
