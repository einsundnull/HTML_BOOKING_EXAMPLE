## What I understand

**The bug:** When student clicks `+ Add` in the free slots accordion:
1. Slot is staged in `pendingDayChanges`
2. `renderDaySlots()` is called — full rebuild
3. The new auto-expand logic sets `_daySlotsFreeOpen = false` because a pending block exists
4. Free slots accordion **collapses** — appears to the user as if slots disappeared
5. The pending block opens showing the staged slot

**The correct behavior:** Stage the slot silently with amber highlight. The free slots accordion should **stay open**. The staged slot should turn amber in-place (like teacher's `onAddSlot` path 2 does via `populateBookingDetail` which just re-renders the detail in-place via `onAction`).

**Root cause:** Student's free slots `+ Add` goes through a completely separate code path (direct DOM row build) instead of going through `populateBookingDetail` like the block-detail path does. The block-detail path works perfectly — amber in-place, no accordion collapse.

**Fix:** The free slots `+ Add` in student `renderDaySlots` should NOT call `renderDaySlots()` — it should only update the specific row in-place (change button to "Undo", add amber class) + call `updateDaySaveBtn()`. Same pattern as `populateBookingDetail`'s `onAction`.
