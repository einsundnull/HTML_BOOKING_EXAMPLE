Mobile Layout

Regeln für mobile Browser-Kompatibilität und responsives Layout. Diese Guideline ergänzt die Styling-Guideline um mobile-spezifische Hard-Rules: Touch-Targets, Safe-Area-Insets, Scroll-Verhalten, iOS-Safari-Fixes und das responsive Verhalten aller UI-Komponenten.

🔴 Hard-Rule 🔵 Soft-Rule ⛔ Verboten iOS Safari · Android Chrome · PWA-ready
🔴 Hard-Rule
Nicht verhandelbar. Bricht Mobile-UX wenn ignoriert.
🔵 Soft-Rule
Starke Empfehlung. Abweichung erfordert Begründung.
🟢 Arch-Rule
Strukturell. CSS-Konvention die immer gilt.
⛔ Verboten
Erzeugt Bugs auf Touch-Geräten oder iOS-Safari.

1. Breakpoints

2 Regeln
🔴 Hard
Fünf Breakpoints — alle durch 8 teilbar — keine eigenen Werte
Keine media queries mit 900px, 1100px, 600px oder anderen nicht auf der Skala liegenden Werten.
TokenWertKontextSection-SpacingContainer-Padding
--bp-sm 480px Kleines Smartphone (Portrait) — var(--sp-2) 8px
--bp-md 768px Tablet (Portrait) / Smartphone (Landscape) var(--sp-6) 48px var(--sp-3) 16px
--bp-lg 1024px Tablet (Landscape) / kleiner Desktop var(--sp-7) 64px var(--sp-4) 24px
--bp-xl 1280px Standard-Desktop var(--sp-8) 96px var(--sp-5) 32px
--bp-2xl 1536px Breiter Screen / externes Display var(--sp-8) 96px var(--sp-5) 32px
Premium · StyleGuide
🔴 Hard
Mobile-First Media Queries mit min-width
Basis-CSS schreibt den mobilen Zustand. Desktop-Overrides kommen mit @media (min-width: ...). Ausnahme: Spezifische Rücknahmen (z.B. Hover-States) dürfen max-width nutzen.
/* ✅ Mobile-First: Basis = Mobile, Desktop per min-width */ .card { padding: var(--sp-3); /* Mobile: 16px */ } @media (min-width: 768px) { .card { padding: var(--sp-4); } /* Tablet+: 24px */ } @media (min-width: 1024px) { .card { padding: var(--sp-5); } /* Desktop: 32px */ } /* ❌ Desktop-First: schlechter Ausgangspunkt für Mobile */ .card { padding: var(--sp-5); } /* Desktop als Basis */ @media (max-width: 768px) { .card { padding: var(--sp-3); } }

2. Touch Targets

3 Regeln
Touch-Target-Größen im Vergleich
24
⛔ Zu klein
36
⚠ Grenzwertig
44px

(Apple HIG)
48px
✅ Ideal
(Material)
🔴 Hard
Alle interaktiven Elemente: min-height: 44px auf Mobile
Apple HIG und WCAG 2.5.5 definieren 44×44px als Mindest-Touch-Ziel. Das gilt für Buttons, Links, Checkboxen, Selects, Nav-Links und alle anderen tappbaren Elemente. Visuell darf das Element kleiner sein — aber der tappbare Bereich muss 44px erreichen.
/* ✅ Richtig: visuell klein, Touch-Area groß */ @media (max-width: 768px) { .btn { min-height: 44px; } .nav-link { min-height: 44px; display: flex; align-items: center; } .form-select { min-height: 44px; } /* Icon-only Buttons: padding statt min-height */ .btn-icon { padding: 10px; } /* 24px icon + 10px * 2 = 44px */ }
StyleGuide · Apple HIG · WCAG 2.5.5
🔴 Hard
touch-action: manipulation auf allen Buttons
Eliminiert den 300ms Tap-Delay auf iOS (Double-Tap-Zoom-Prävention). Gilt für alle .btn-Varianten, Nav-Links und alle tappbaren Custom-Elemente.
.btn, .nav-link, [role="button"] { touch-action: manipulation; -webkit-tap-highlight-color: transparent; }
Premium · iOS-Fixes
🔴 Hard
Hover-States nur bei echten Zeigegeräten
Ohne diese Media-Query "klebt" der Hover-State nach einem Tap auf Touch-Geräten dauerhaft. :active übernimmt das visuelle Touch-Feedback ohne Media-Query.
✅ @media (hover: hover) and (pointer: fine) { .btn:hover { background: var(--accent-hover); } } ❌ .btn:hover { background: var(--accent-hover); } /* klebt nach Tap auf Touch — verboten */ /* :active braucht keine Media-Query — funktioniert auf Touch */ ✅ .btn:active { background: var(--color-600); }
Premium · iOS-Fixes

3. iOS Safari — Pflicht-Fixes

6 Regeln
iOS Safari verhält sich in mehreren Punkten anders als alle anderen Browser. Diese Fixes müssen im globalen Reset (base.css) vorhanden sein — sie sind keine optionalen Polyfills sondern notwendige Korrekturen.
🔴 Hard
viewport-fit=cover im Meta-Tag
Ohne diesen Parameter liefert env(safe-area-inset-*) immer 0 zurück. iPhone Notch und Dynamic Island werden dann nicht berücksichtigt.
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
iOS-Fixes #1
🔴 Hard
Safe-Area-Insets für Notch / Dynamic Island
Navbar und Footer müssen die Safe Areas berücksichtigen, damit Inhalte nicht unter der Notch oder dem Home-Indicator verschwinden.
/* Im Reset: Safe-Area-Tokens definieren */ :root { --safe-top: env(safe-area-inset-top, 0px); --safe-bottom: env(safe-area-inset-bottom, 0px); --safe-left: env(safe-area-inset-left, 0px); --safe-right: env(safe-area-inset-right, 0px); } /* Navbar: oben geschützt */ .navbar { padding-top: var(--safe-top); } /* Footer / fixed bottom bar: unten geschützt */ .footer-fixed { padding-bottom: calc(var(--sp-3) + var(--safe-bottom)); }
iOS-Fixes #7
🔴 Hard
-webkit-text-size-adjust: 100% auf html
Verhindert automatische Schriftgrößenanpassung bei Landscape-Rotation in iOS Safari. Gehört in den globalen Reset.
html { -webkit-text-size-adjust: 100%; text-size-adjust: 100%; /* Standard für zukünftige Browser */ }
iOS-Fixes #2
🔴 Hard
-webkit-tap-highlight-color: transparent im Reset
Entfernt den blauen/grauen Flash bei jedem Tap auf iOS und Android. Ohne dieses Reset erscheint bei jedem Button-Tap ein hässlicher Highlight-Effekt.
* { -webkit-tap-highlight-color: transparent; }
iOS-Fixes #4
🔴 Hard
position: -webkit-sticky als Fallback für Sticky-Elemente
iOS Safari benötigt vor Version 13 das Webkit-Prefix. Beide Varianten immer zusammen schreiben.
.navbar { position: -webkit-sticky; /* iOS Safari < 13 */ position: sticky; top: 0; }
iOS-Fixes #6
🔴 Hard
:focus-visible Fallback für Safari < 15.4
iOS Safari unterstützt :focus-visible erst ab Version 15.4. Für ältere Versionen mit @supports not als Fallback absichern.
/* Primär: :focus-visible (moderne Browser) */ .btn:focus-visible { outline: none; box-shadow: var(--focus-ring); } /* Fallback: :focus für Safari < 15.4 */ @supports not selector(:focus-visible) { .btn:focus { outline: none; box-shadow: var(--focus-ring); } }
iOS-Fixes #11

4. Scroll & Overflow

4 Regeln
🔴 Hard
Horizontaler Scroll auf der Seite: immer verhindern
overflow-x: hidden auf body ist der erste Schutzmechanismus. Ursache ist fast immer ein Element das breiter als 100vw wird — häufig Code-Blöcke oder Tabellen ohne Wrapper.
body { overflow-x: hidden; } /* Horizontale Scroll-Container explizit erlauben */ .table-wrapper, .code-scroll { overflow-x: auto; -webkit-overflow-scrolling: touch; /* Momentum Scroll auf iOS */ }
StyleGuide · iOS-Fixes
🔴 Hard
Drawers und Modals: Body-Scroll sperren beim Öffnen
Wenn ein Modal oder Drawer offen ist, muss Scrollen auf der Hauptseite verhindert werden. Die einfachste zuverlässige Methode auf iOS:
/* Body-Scroll sperren */ function lockScroll() { document.body.style.overflow = 'hidden'; } function unlockScroll() { document.body.style.overflow = ''; } /* WICHTIG: bei Drawer/Modal open/close aufrufen */ MyDrawer.open → lockScroll() MyDrawer.close → unlockScroll()
Layout · StyleGuide
🔴 Hard
overscroll-behavior: contain in scrollbaren Panels
Verhindert dass Scroll aus einem Drawer/Modal auf die Hauptseite überläuft (Scroll-Chaining). Besonders auf iOS und Android wichtig.
.drawer-content, .modal-body, .scrollable-panel { overflow-y: auto; overscroll-behavior: contain; -webkit-overflow-scrolling: touch; }
StyleGuide
🔵 Soft
Smooth Scroll nur wenn keine reduced-motion Präferenz
iOS Safari unterstützt scroll-behavior: smooth erst ab Version 15.4. Immer in die reduced-motion Media-Query einbetten.
@media (prefers-reduced-motion: no-preference) { html { scroll-behavior: smooth; } }
iOS-Fixes #3 · StyleGuide

5. Responsive Komponenten

5 Regeln
🔴 Hard
Navbar auf Mobile: Hamburger-Menü — keine horizontale Linkliste
Ab 768px werden Desktop-Navbar-Links ausgeblendet und durch ein Hamburger-Icon mit Drawer ersetzt. Der Drawer öffnet sich von oben oder von der Seite, nimmt die volle Breite ein und sperrt den Body-Scroll.
/* Pattern: Desktop-Links verstecken, Toggle zeigen */ .nav-links { display: flex; } .nav-toggle { display: none; } @media (max-width: 768px) { .nav-links { display: none; } .nav-toggle { display: flex; } .nav-mobile { /* Drawer: full-width, fixed */ } }
StyleGuide · Layout
🔴 Hard
Tabellen: horizontaler Scroll-Wrapper auf Mobile
Tabellen werden auf kleinen Screens nicht gequetscht — sie bekommen einen scrollbaren Wrapper. Keine table-layout: fixed auf Mobile ohne Scroll-Container.
/* Immer: Tabelle in Wrapper einschließen */ <div class="table-wrapper"> <table class="table">...</table> </div> .table-wrapper { overflow-x: auto; -webkit-overflow-scrolling: touch; border-radius: var(--radius-md); border: 1px solid var(--border); }
Layout
🔴 Hard
Form-Inputs: font-size ≥ 16px verhindert iOS-Zoom
iOS Safari zoomt automatisch hinein wenn ein Input den Fokus erhält und font-size kleiner als 16px ist. Alle input, textarea, select müssen mindestens 16px haben.
input, textarea, select { font-size: var(--text-body); /* 16px — verhindert iOS-Zoom */ } /* ❌ Führt zu ungewolltem Zoom auf iOS: */ input { font-size: 14px; }
iOS-Spezifikation
🔴 Hard
Drawers: transform statt left/right/top animieren
Layout-Properties (left, top) triggern Layout-Berechnungen — performanceintensiv auf Mobile. Immer transform: translateX() für Drawer-Animationen.
.drawer { position: fixed; right: 0; top: 0; bottom: 0; width: min(480px, 100vw); /* max 480px, aber nie breiter als Viewport */ transform: translateX(100%); /* Hidden: aus dem Viewport */ transition: transform 200ms ease; will-change: transform; } .drawer.is-open { transform: translateX(0); /* Sichtbar */ }
Premium · StyleGuide
🔵 Soft
Cards auf Mobile: volle Breite statt Grid
Card-Grids die auf Desktop 3 oder mehr Spalten haben, reduzieren auf Mobile auf 1 Spalte. Tablet (≥ 768px): 2 Spalten. Desktop (≥ 1024px): Original-Grid.
.card-grid { display: grid; grid-template-columns: 1fr; /* Mobile: 1 Spalte */ gap: var(--sp-3); } @media (min-width: 768px) { .card-grid { grid-template-columns: repeat(2, 1fr); } } @media (min-width: 1024px) { .card-grid { grid-template-columns: repeat(3, 1fr); } }
StyleGuide

6. Logical Properties & RTL

2 Regeln
🔵 Soft
Logical Properties bevorzugen + physikalischen Fallback davor
padding-inline, margin-block etc. sind RTL-fähig. iOS 14.0 unterstützt sie vollständig, darunter braucht es Fallbacks.
/* Richtig: physikalischer Fallback + logical override */ .container { padding-left: var(--sp-5); /* Fallback */ padding-right: var(--sp-5); padding-inline: var(--sp-5); /* Überschreibt bei Support */ }
STYLE-GUIDE · iOS-Fixes #9
🔵 Soft
CSS Grid: grid-gap und gap zusammen schreiben
gap in CSS Grid ist erst ab iOS 12 stabil. Für breitere Kompatibilität beide Properties schreiben.
.grid { grid-gap: var(--sp-3); /* Fallback iOS < 12 */ gap: var(--sp-3); /* Standard */ }
iOS-Fixes #10

7. Vollständiger Mobile-Reset (base.css)

Dieser Block gehört in base.css — er muss auf jeder Seite geladen werden. Er fasst alle Pflicht-Fixes aus den vorherigen Sektionen zusammen.
/* ════════════════════════════════════════════════════════ Mobile Reset — base.css Alle Pflicht-Fixes für Mobile Browser ════════════════════════════════════════════════════════ */ /* 1. Box model + Tap-Highlight */ *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; -webkit-tap-highlight-color: transparent; } /* 2. iOS Safari text-size-adjust */ html { -webkit-text-size-adjust: 100%; text-size-adjust: 100%; } /* 3. Smooth Scroll nur ohne reduced-motion */ @media (prefers-reduced-motion: no-preference) { html { scroll-behavior: smooth; } } /* 4. Body: no horizontal overflow */ body { overflow-x: hidden; min-height: 100vh; min-height: -webkit-fill-available; /* iOS 100vh fix */ } /* 5. Touch-Targets: Buttons, Links */ button, [role="button"], a, input, select, textarea { touch-action: manipulation; } /* 6. Inputs: font-size ≥ 16px → kein iOS-Zoom */ input, textarea, select { font-size: max(16px, 1rem); } /* 7. Sticky: Webkit-Prefix Fallback */ .is-sticky { position: -webkit-sticky; position: sticky; } /* 8. Focus-Visible Fallback für Safari < 15.4 */ @supports not selector(:focus-visible) { *:focus { outline: none; box-shadow: var(--focus-ring); } } /* 9. Safe-Area-Insets (benötigt viewport-fit=cover im meta-Tag) */ :root { --safe-top: env(safe-area-inset-top, 0px); --safe-bottom: env(safe-area-inset-bottom, 0px); --safe-left: env(safe-area-inset-left, 0px); --safe-right: env(safe-area-inset-right, 0px); }

8. Verbotene Mobile-Muster

5 Einträge
⛔ Verboten
Hover-States ohne @media (hover: hover)
Klebt nach dem Tap dauerhaft. Jeder :hover-State muss in die hover-Media-Query eingeschlossen sein.
iOS-Fixes #5
⛔ Verboten
Input font-size unter 16px
Triggert automatischen iOS-Zoom beim Fokus. Immer max(16px, 1rem) für Inputs.
iOS-Spezifikation
⛔ Verboten
overflow: hidden auf einem Scroll-Container-Parent
Bricht position: sticky für alle Kinder. Immer overflow: clip prüfen oder das Layout umstrukturieren.
iOS-Fixes #12
⛔ Verboten
Tabellen ohne horizontalen Scroll-Wrapper
Erzeugt horizontalen Page-Scroll. Jede Tabelle braucht einen .table-wrapper mit overflow-x: auto.
Layout
⛔ Verboten
Layout-Animationen (left, top, width) statt transform
Triggert Layout-Recalculations und ist auf Mobile-CPUs deutlich langsamer. Immer transform und opacity.
