Styling & Layout

Alle visuellen Regeln dieses Projekts — von der Farbe bis zum Grid. Diese Datei ersetzt UI-Guide Lines Premium, Guideline_Layout_and_Styling, STYLE-GUIDE, Styling-Guide und STYLE-DESCRIPTION. Hard-Rules sind nicht verhandelbar. Soft-Rules erfordern eine Begründung bei Abweichung.

🔴 Hard-Rule 🔵 Soft-Rule ⛔ Verboten Stand: März 2026
🔴 Hard-Rule
Nicht verhandelbar. Immer einhalten. Keine Ausnahmen ohne Architekturentscheidung.
🔵 Soft-Rule
Starke Empfehlung. Abweichung erfordert eine dokumentierte Begründung.
🟢 Arch-Rule
Strukturell unveränderlich. Betrifft Dateien, Schichten und Konventionen.
⛔ Verboten
Explizit verboten. Bricht Konsistenz, Wartbarkeit oder das Token-System.

1. Typografie

5 Regeln
🔴 Hard
Font: ausschließlich Figtree — kein zweiter Font
font-family: 'Figtree', sans-serif ist der einzige erlaubte Font im gesamten Projekt. Kein zweiter Schriftzug, kein System-Font außerhalb von Monospace-Ausnahmen (Code-Blöcke: JetBrains Mono).
✅ --font: 'Figtree', sans-serif; ❌ font-family: Inter, Arial, system-ui; /* verboten */
Premium · Layout · StyleGuide
🔴 Hard
Nur 2 Gewichte: 400 Regular + 600 Semibold
Das 4-2-Prinzip. Drittes Gewicht (z.B. 500 Medium) ist verboten — auch nicht über opacity oder filter annähern.
✅ --weight-regular: 400; /* Fließtext, Captions */ ✅ --weight-semibold: 600; /* Headlines, CTAs, Nav */ ❌ font-weight: 500; /* verboten */
Premium · StyleGuide
🔴 Hard
Genau 4 Schriftgrößen — alle durch 4 teilbar
Display 48px · Heading 24px · Body 16px · Caption 12px. Keine Zwischengrößen (z.B. 14px, 18px, 20px). Alle Größen via Token.
--text-display: 48px /* Hero, Seitentitel */ --text-heading: 24px /* Section-Titel, Card-Titel */ --text-body: 16px /* Fließtext, Beschreibungen */ --text-caption: 12px /* Labels, Metadaten, Timestamps */
Premium · Layout
🔴 Hard
Schriftgrößen immer als CSS-Token — niemals hardcoded
Jede Schriftgröße im CSS referenziert einen Token. Keine font-size: 16px direkt im Component-CSS.
STYLE-GUIDE · Layout
🔴 Hard
iOS: -webkit-text-size-adjust: 100% auf html
Verhindert automatische Schriftgrößenanpassung im iOS-Safari bei Landscape-Rotation. Gehört in den globalen Reset.

6 Regeln

🔴 Hard
Nur eine Hue: Navy #060f1c — keine zweite Hue
Die gesamte Farbpalette besteht aus Tints und Shades dieser einen Basis. Kein zweites Blau, kein Grün für Akzente, keine branchenfremden Farben.
Premium · StyleGuide
🔴 Hard
Alle Farben als CSS-Tokens — niemals Hex-Werte direkt im Component-CSS
Hex-Werte nur in tokens.css innerhalb der :root-Definition. Jede andere CSS-Datei referenziert ausschließlich Tokens.
✅ color: var(--text-primary); ❌ color: #0d1117; /* hardcoded — verboten */

Tiefe entsteht durch Spacing und Kontrast — nicht durch Farbverläufe oder Texturen. Shadows sind erlaubt, aber subtil: 0 1px 3px rgba(6,15,28,0.08).

Kontrast ≥ 4.5:1 für Fließtext (WCAG AA)
Headings nie grau. Muted-Text (neutral-500) nur für sekundäre, nicht-kritische Inhalte. Basis-Textfarbe: neutral-900 oder color-900.

Eigene Token-Skala (--danger-*, --success-*) in tokens.css. Diese Farben abgeleitet aus bestehenden Rot-/Grün-Werten halten — nicht eigenständige neue Hues einführen.

3. Spacing — 8pt Grid

4 Regeln
🔴 Hard
Alle Abstände durch 4 oder 8 teilbar — keine Ausnahmen
15px, 22px, 7px, 13px sind verboten. Jeder Margin, Padding, Gap im gesamten CSS muss auf der Skala liegen.
TokenWertVerwendung
--sp-14pxIcon-Padding, enge inline Gaps
--sp-28pxLabel ↔ Icon, Input-Padding
--sp-316pxElemente innerhalb einer Gruppe
--sp-424pxCard-internes Padding
--sp-532pxZwischen Gruppen / Komponenten
--sp-648pxSection-Padding (Mobile)
--sp-764pxSection-Padding (Desktop)
--sp-896pxGroße Layout-Trennung
--sp-9128pxHero-Vertikalrhythmus
Premium · Layout · StyleGuide
🔴 Hard
Proximity-Prinzip: Abstand kommuniziert Beziehung
16px = gleiche Gruppe (Label über Input, Icon neben Text). 32px = verschiedene Gruppen. 64px+ = verschiedene Sections. Das Auge liest Nähe als Zusammengehörigkeit — diese Logik muss konsistent eingehalten werden.
Premium
🔴 Hard
Breakpoints durch 8 teilbar: 480 / 768 / 1024 / 1280 / 1536 px
Alle Media-Query-Breakpoints liegen auf der 8pt-Skala. Keine beliebigen Werte wie 900px oder 1100px.
Premium · StyleGuide
🔵 Soft
Logical Properties + physikalischer Fallback für RTL-Support
padding-inline statt padding-left/right. Immer physikalische Property zuerst als Fallback für ältere Browser, dann logical überschreiben.
/* Richtig: Fallback + Logical */ padding-left: var(--sp-5); padding-right: var(--sp-5); padding-inline: var(--sp-5); /* überschreibt bei Support */
STYLE-GUIDE

4. Layout

4 Regeln
🔴 Hard
12-Column Grid. Container max-width 1280px, padding 32px.
Content in 8 von 12 Spalten auf Desktop. Hero-Text zentriert in 8 Spalten. Volle Breite = 12/12, Content + Sidebar = 8+4, zwei gleiche = 6+6.
.container { width: 100%; max-width: var(--container-max); /* 1280px */ margin-inline: auto; padding-inline: var(--sp-5); /* 32px */ }
Premium · StyleGuide
🔴 Hard
Standard-Seitenstruktur: app-shell → main.page → page-header
Jede Seite folgt diesem Template. Navbar wird via Navbar.init() injiziert. Kein Abweichen von dieser Grundstruktur.
<div class="app-shell"> <!-- Navbar via Navbar.init() --> <main class="page"> <div class="page-header"> <h2>Seitentitel</h2> <p>Untertitel</p> </div> <!-- Seiteninhalt --> </main> </div>
Layout
🔴 Hard
Navbar-Höhe: 64px. Immer sticky, border-bottom 1px, bg: neutral-0.
Alle Seiten nutzen Navbar.init(pagename, options). Keine eigene Navbar pro Seite implementieren.
Layout · Premium
🔴 Hard
Sichtbarkeit via CSS-Klasse — niemals via inline JS
Elemente ein-/ausblenden ausschließlich über .is-hidden (display:none) und .view-hidden.
✅ el.classList.add('is-hidden'); ✅ el.classList.remove('is-hidden'); ❌ el.style.display = 'none'; /* verboten */ ❌ el.style.visibility = 'hidden'; /* verboten */

5. Komponenten

5 Regeln
🔴 Hard
Jedes interaktive Element braucht alle 5 Zustände
default · hover · active · focus · disabled — kein Element ohne vollständige State-Abdeckung in Produktion.
.btn-primary { background: var(--accent); } .btn-primary:hover { background: var(--accent-hover); } .btn-primary:active { background: var(--color-600); } .btn-primary:focus-visible { box-shadow: var(--focus-ring); } .btn-primary:disabled { opacity: 0.4; cursor: not-allowed; }
Premium
🔴 Hard
Focus-Ring: immer sichtbar, niemals unterdrückt
outline: none nur in Kombination mit einem sichtbaren box-shadow-Ersatz. :focus-visible verwenden — mit Fallback via @supports not selector(:focus-visible).
Premium · StyleGuide
🔴 Hard
Hover-States nur in @media (hover: hover) and (pointer: fine)
Verhindert "sticky hover" auf Touch-Geräten. Active-State (:active) übernimmt Touch-Feedback ohne Media-Query.
✅ @media (hover: hover) and (pointer: fine) { .btn:hover { background: var(--accent-hover); } } ❌ .btn:hover { ... } /* direkt — klebt auf Touch */
Premium · iOS-Fixes
🔴 Hard
Nur eine Icon-Library — Größen 16px oder 24px
Keine Emojis als Icons. Keine gemischten Icon-Styles. Primäre Aktionen: Icon immer mit Textlabel kombiniert.
Premium · StyleGuide
🔵 Soft
Touch Targets: min-height 44px für alle Buttons und Links
Apple HIG und WCAG empfehlen 44×44px als Mindestgröße für Touch-Ziele. Auf Mobile immer prüfen.
StyleGuide

6. Motion & Übergänge

4 Regeln
🔴 Hard
Transitions: 120ms ease (Hover) · 200ms ease (Panels/Drawers)
Motion kommuniziert State-Änderung — niemals Persönlichkeit. Kein Bounce, kein Spring, kein Elastic.
--transition-fast: 120ms ease; /* Hover, Icon-Wechsel */ --transition-base: 200ms ease; /* Panels, Drawers, Modals */
Premium
🔴 Hard
Keine Layout-Animationen — nur transform und opacity
width, height, top, left animieren = Layout-Thrashing. Immer transform: translateX() bzw. opacity für Übergänge.
Premium
🔵 Soft
Smooth Scroll nur in prefers-reduced-motion: no-preference
Scroll-Animationen respektieren Accessibility-Einstellungen des Betriebssystems.
StyleGuide
🔵 Soft
CTAs: max. 3 Wörter, Verb-First, keine Füllwörter
"Anmelden", "Speichern", "Weiter" statt "Bitte hier klicken um fortzufahren". Passive Formulierungen vermeiden: "Gespeichert" statt "Wurde erfolgreich gespeichert".
Premium · Styling-Guide

7. Verbotene Praktiken

5 Einträge
Diese Muster werden in Code-Reviews abgelehnt und müssen vor jedem Merge behoben werden.
⛔ Verboten
Inline-CSS im HTML (style="...")
Bricht das Token-System und macht Theming unmöglich. Einzige Ausnahme: dynamische Werte die per JS zur Laufzeit gesetzt werden müssen (z.B. dynamische Breiten).
Layout · GUIDELINES_JS
⛔ Verboten
Hardcoded Hex/px-Werte außerhalb von :root in tokens.css
Keine color: #4d7aa0 oder padding: 24px direkt in Component-CSS-Dateien.
Layout · STYLE-GUIDE
⛔ Verboten
window.alert() / window.confirm() / window.prompt()
Browser-Dialoge sind nicht themebar und blockieren den Rendering-Thread. Immer Modal.show() oder Toast.* aus ui.js.
Layout
⛔ Verboten
Mehr als 2 Font-Gewichte oder mehr als eine Hue
Bricht das Premium-Grundprinzip. Auch 500 Medium simuliert über opacity ist verboten.
Premium
⛔ Verboten
Gradients, Texturen, starke Schatten für Tiefeneffekte
Tiefe entsteht ausschließlich durch Spacing-Hierarchie und Farbkontrast. Shadow max: 0 1px 3px rgba(6,15,28,0.08).
