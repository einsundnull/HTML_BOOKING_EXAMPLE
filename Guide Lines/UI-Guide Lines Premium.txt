  2. Color System

  Based on Eric D. Kennedy's single-hue scale methodology and the 60-30-10 rule.

  Core principle: Never mix unrelated hues. Build all color from one base using tints (add white) and shades (add black), adjusting saturation as lightness changes.

  Base color: #060f1c — HSL 216, 65%, 7% — deep navy

  Tint & Shade Scale

  Lighter tints reduce saturation. Darker shades hold or slightly increase it. This keeps the scale from looking washed out at the top or muddy at the bottom.

  Max 4 sizes. Max 2 weights.

  ├──────────────┼──────┼─────────────┼──────────────┼──────────────────────────────┤

  │ text-body    │ 16px │ 24px        │ 400 Regular  │ Body copy, descriptions      │
  ├──────────────┼──────┼─────────────┼──────────────┼──────────────────────────────┤
  │ text-caption │ 12px │ 16px        │ 400 Regular  │ Labels, metadata, timestamps │
  └──────────────┴──────┴─────────────┴──────────────┴──────────────────────────────┘

  All sizes are divisible by 8 or 4.

  Weight usage:
  - 400 Regular — body, captions, secondary content
  - 600 Semibold — headings, CTAs, navigation, emphasis

  Never use 3 weights. Never approximate a third weight by using opacity.

  4. Spacing — 8pt Grid System

  Every spacing value must be divisible by 8 or 4. No exceptions.

  Base Scale

  ┌─────────┬───────┬───────────────────────────────────────┐
  │  Token  │ Value │                  Use                  │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-1 │ 4px   │ Icon padding, tight inline gaps       │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-2 │ 8px   │ Between label and icon, input padding │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-3 │ 16px  │ Between elements within one group     │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-4 │ 24px  │ Internal card padding                 │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-5 │ 32px  │ Between separate groups / sections    │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-6 │ 48px  │ Section top/bottom padding (mobile)   │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-7 │ 64px  │ Section top/bottom padding (desktop)  │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-8 │ 96px  │ Major layout separation               │
  ├─────────┼───────┼───────────────────────────────────────┤
  │ space-9 │ 128px │ Hero vertical rhythm                  │
  └─────────┴───────┴───────────────────────────────────────┘

  Grouping Rule

  The spacing between elements communicates their relationship. Use it consistently:

  - 16px — elements within the same group (label above input, icon next to text)
  - 32px — between distinct groups or components (one card to another, one form section to the next)
  - 64px+ — between page-level sections

  The eye reads proximity as relationship. If two things are 32px apart they belong to the same logical area. If they are 64px apart they are in different sections.

  Application Examples

  Button padding:    16px vertical,  24px horizontal
  Card padding:      24px all sides
  Form field gap:    16px (same group)
  Form section gap:  32px (different group)
  Nav item gap:      24px
  Section gap:       64px
  Container max-w:   1280px, centered

  5. Premium Design Patterns (Researched)

  These are consistent across Stripe, Linear, Vercel, Apple, Framer:

  Layout
  - One primary action per section. Never compete with yourself.
  - Left-aligned text is more readable than centered body copy. Reserve center-alignment for short hero headlines only.
  - Use a 12-column grid. Content lives in 8 of those columns on desktop.

  Visual Weight
  - High contrast between text and background. Never drop below 4.5:1 for body text.
  - Use neutral-900 or color-900 for headings. Never grey for primary headings.
  - Muted text (neutral-500) only for truly secondary content.

  Surfaces & Depth
  - No drop shadows with heavy blur. If you use a shadow, keep it subtle: 0 1px 3px rgba(6,15,28,0.08).
  - Cards: a 1px border in neutral-200 reads cleaner than a shadow on white.
  - Flat backgrounds. Depth comes from spacing and contrast, not texture or gradient.

  Interactive States
  - Every interactive element needs: default, hover, active, focus, disabled.
  - Focus ring: 2px solid color-400 with 2px offset — always visible, never suppressed.
  - Hover: shift one step down the color scale (color-400 → color-500).

  Icons
  - Use a single icon library. Never mix styles.
  - Size: 16px or 24px. Both are divisible by 8.
  - Icons are always paired with a text label on primary actions.

  8. Component Specifications

  Button

  Three variants — primary, secondary, ghost. One size per context.

  .btn {
    font-family: var(--font-family);
    font-size: var(--text-body);
    font-weight: var(--weight-semibold);
    line-height: var(--leading-body);
    display: inline-flex;
    align-items: center;
    gap: var(--space-2);
    padding: var(--space-2) var(--space-4);   /* 8px 24px */
    border-radius: var(--radius-md);
    border: 1px solid transparent;
    cursor: pointer;
    transition: background var(--duration-fast) var(--easing-base),
                border-color var(--duration-fast) var(--easing-base);
    text-decoration: none;
    white-space: nowrap;
  }

  /* Primary — 10% brand */
  .btn-primary {
    background: var(--accent);
    color: var(--neutral-0);
    border-color: var(--accent);
  }
  .btn-primary:hover  { background: var(--accent-hover); border-color: var(--accent-hover); }
  .btn-primary:active { background: var(--accent-active); border-color: var(--accent-active); }

  /* Secondary — 30% complementary */
  .btn-secondary {
    background: var(--neutral-0);
    color: var(--text-primary);
    border-color: var(--border-strong);
  }
  .btn-secondary:hover  { background: var(--bg-subtle); }
  .btn-secondary:active { background: var(--neutral-200); }

  /* Ghost — no fill */
  .btn-ghost {
    background: transparent;
    color: var(--text-secondary);
    border-color: transparent;
  }
  .btn-ghost:hover  { background: var(--bg-subtle); color: var(--text-primary); }
  .btn-ghost:active { background: var(--neutral-200); }

  /* Disabled — all variants */
  .btn:disabled,
  .btn[aria-disabled="true"] {
    opacity: 0.4;
    cursor: not-allowed;
    pointer-events: none;
  }

  /* Focus — all variants */
  .btn:focus-visible {
    outline: none;
    box-shadow: var(--focus-ring);
  }

  Copy rules for buttons:
  - Primary: action verb + noun if needed (Get started, Save changes, Continue)
  - Secondary: explicit but lighter (Cancel, Go back, Learn more)
  - Ghost: navigation or tertiary (View all, Skip)

  11. Transitions & Motion

  Premium interfaces move quietly. Motion communicates state change — never personality.

  /* Use these only */
  --duration-fast: 120ms;   /* hover states, icon swaps */
  --duration-base: 200ms;   /* panels, dropdowns */
  --easing-base: ease;

  Rules:
  - Color and opacity transitions: 120ms ease
  - Elements entering the screen (modals, drawers): 200ms ease
  - Never animate layout properties (width, height, top) — animate transform and opacity instead
  - No bounce, spring, or elastic easings — these undermine credibility

  ---
  This completes the full system. Every value traces back to a token. Every token traces back to a rule. That traceability is what keeps a premium interface consistent as it grows.
